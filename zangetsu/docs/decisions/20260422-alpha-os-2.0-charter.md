## 1. Mission

Transform Zangetsu from a single-route alpha search system into an **Alpha Operating System** that can:

1. recover and stabilize the current production pipeline,
2. preserve and formalize validated routing / policy assets,
3. discover real information edge through a gated research process,
4. route, evaluate, and eventually allocate capital across validated alpha programs.

This charter starts from the current confirmed reality: production recovery is not complete, and the old GP / current LGBM route is not a viable main alpha source under the current formulation.

---

## 2. Strategic Principles

### 2.1 Recovery comes before discovery
No research result may be promoted into the production mainline until the production blocker path is unblocked.

### 2.2 Do not keep optimizing a failed formulation
The following has already failed as a main alpha route:
- GP mainline
- archive champions under current conditions
- Phase 3E GP re-architecture
- LGBM baseline on the same formulation.

### 2.3 Preserve the execution core
Keep and reuse:
- data ingestion / parquet store
- indicator cache
- backtester
- V10 gates
- staging / fresh / rejected / telemetry
- policy layer
- family-aware routing
- exception overlay

### 2.4 Discovery must be gated
All future alpha work must follow:
**Target Gate → Feature Gate → Model Gate → Deployment Gate**

### 2.5 No mixed patches
Never combine: production hotfixes, policy stabilization, discovery experiments, canary deployment into the same patch or release unit.

R2 Deliverables: patch application log; migration log; before/after metrics table; Arena2 status delta; validator status delta; rollback artifacts.

### Phase S1 — Policy / Routing Asset Freeze (after G2 or fully isolated parallel)
Scope: Policy Layer v0, family-aware routing, exception overlay, replay/audit tools, ADRs, decision memos, retrospective bundle.
Acceptance: commit-ready policy/resolver/overlay; active/fallback/blocked/exception states fully defined; ADR coverage for major decisions; expiry/review mechanisms documented and testable.

### Phase D0 — Discovery Readiness Gate
Must answer: production stable enough for baseline? discovery artifacts real? D1 scoped clearly?
Acceptance: D1 artifact plan exists; discovery isolated from production; recovery is no longer dominant blocker.

### Phase D1 — Upstream Signal Quality Audit
Objective: audit whether current formula space and primitive envelope are structurally too noisy.
Required outputs: primitive usage distribution; terminal usage distribution; formula complexity distribution; train_neg_pnl dominant patterns; feature SNR analysis; directional-prior assessment of primitive set.

### Phase D2 — Target Gate
Mandatory target candidates: triple-barrier hit probability; vol-normalized direction; regime-conditional binary move; event-response target.
Required metrics: rank IC / rank IC stability; MI / information gain; AUC / hit-rate; permutation robustness; train→holdout stability; regime stratification.
Rule: any target with no holdout edge is eliminated.

### Phase D3 — Feature Gate
Feature tiers:
- T1 Cheap: funding, OI delta, basis, liquidation proxy, cross-symbol, regime context
- T2 Medium: perp/spot divergence, cross-exchange spread, sector-relative flow, volatility composites
- T3 Expensive: order-book imbalance, queue imbalance, depth slope, aggressor flow, tick event streams
Rule: T1 must show signal before T3 build is allowed.

### Phase D4 — Model Gate
Required sequence: logistic/linear → LightGBM → MLP → sequence/Transformer.
Rule: no complex model before cheap baselines succeed.
Acceptance: at least one baseline must produce `inserted>0` under prod-cost conditions, materially outperform GP/old baseline, survive holdout.

### Phase D5 — Alpha Registry / Router / Allocator
- D5-A Alpha Registry: each alpha stores target, feature bucket, model family, direction sign, train/holdout IC, pnl/sharpe/wilson, shrinkage, regime sensitivity, symbol breadth, cost sensitivity.
- D5-B Router: route by regime, target class, feature class, symbol class.
- D5-C Allocator: allocate by reliability, correlation, uncertainty, capacity, cost sensitivity.
