### 專案方向一：動態投資組合最佳化 (Dynamic Portfolio Optimization)

1.  **名稱和一句話描述:** **動態投資組合最佳化 (Dynamic Portfolio Optimization)** - 建立一個基於市場情境、動態調整 13 個 Crypto futures 符號間權重與曝險的智慧型投資組合管理層。

2.  **為什麼適合 j13:**
    *   **自主執行系統:** 最佳化模組將在 Calcifer/Markl 的監督下自主運行，根據 Zangetsu 產生的策略信號和市場情境動態調整。
    *   **Zangetsu V6 復用:** 直接利用 Zangetsu V6 產出的各符號策略信號作為輸入，將分散的單一符號策略提升至投資組合層面。
    *   **Regime-aware:** 投資組合最佳化將深度整合現有的 4h/1h/1m regime-aware 架構，確保在不同市場情境下採用最合適的權重分配模型。
    *   **穩定先於擴展:** 透過分散風險和最佳化資源配置，提升整體投資組合的穩定性和風險調整後收益，而非盲目增加新策略或新資產。
    *   **Occam's Razor:** 專注於優化現有 13 個符號的互動，而非引入新的複雜數據源或交易品種。
    *   **已有 13 symbols:** 這是將單一符號策略的成果，提升到多符號協同作戰的自然演進。

3.  **核心技術架構:**
    *   **Zangetsu 信號層:** 輸出 13 個符號的買/賣/持有信號及預期勝率/潛在盈虧。
    *   **AKASHA 記憶系統:** 儲存歷史策略表現、各符號間的相關性矩陣、市場情境標籤及其對應的最佳權重配置模型。
    *   **Calcifer/Markl 監督層:** 監控市場情境變化，並觸發投資組合最佳化模組。Calcifer 負責學習不同情境下的最佳權重分配策略，Markl 負責執行。
    *   **情境感知最佳化模組 (Python/Numba):**
        *   **輸入:** Zangetsu 信號、當前市場情境 (4h/1h/1m)、各符號實時價格、波動率、相關性矩陣。
        *   **核心邏輯:** 根據當前情境，從 AKASHA 記憶中檢索或由 Calcifer 生成最佳化模型 (例如：Markowitz 變體、風險平價、最大化夏普比率等)，考慮因素包括：
            *   各策略的預期 Alpha (來自 Zangetsu 的勝率/盈虧預期)。
            *   各符號的波動率與相關性。
            *   整體投資組合的目標風險水平。
            *   資金利用率與槓桿限制。
        *   **輸出:** 13 個符號的目標權重 (或頭寸大小)。
    *   **執行層:** 將目標權重轉換為實際交易指令，並透過 Binance API 執行。

4.  **可從 Zangetsu 復用什麼 vs 需要新建什麼:**
    *   **復用:**
        *   Zangetsu V6 的策略生成與回測框架 (作為信號源)。
        *   Zangetsu 的數據管道 (1m bars, 13 symbols)。
        *   Rust indicators, Numba 加速計算能力。
        *   PostgreSQL 數據庫 (儲存最佳化模型的歷史結果、情境標籤)。
        *   Docker 部署環境。
        *   AKASHA 記憶系統與 Calcifer/Markl 自我演化 agent (作為智慧核心)。
        *   Regime-aware 三層架構 (情境識別)。
    *   **新建:**
        *   **情境感知投資組合最佳化模組:** 這是核心，需要設計最佳化演算法、目標函數、約束條件。
        *   **實時相關性/波動率計算模組:** 需要高效計算 13 個符號間的動態相關性與波動率。
        *   **權重分配策略學習模組 (Calcifer 擴展):** Calcifer 需要學習在不同情境下，哪種最佳化模型或參數組合效果最好。
        *   **投資組合級別的 holdout validation 框架:** 評估最佳化模組在未見數據上的表現，防止投資組合層面的過擬合。

5.  **預估複雜度和時間:**
    *   **複雜度:** 中等偏高。核心最佳化演算法的設計和與情境的結合是關鍵。
    *   **時間:** 3-4 個月。
        *   設計與開發核心最佳化模組: 1.5 個月。
        *   整合情境識別與 Calcifer 學習: 1 個月。
        *   回測、holdout validation 與優化: 1 個月。
        *   部署與監控: 0.5 個月。

6.  **主要風險:**
    *   **過擬合 (Portfolio-level Overfitting):** 投資組合最佳化本身也可能過擬合歷史數據，尤其是在情境切換時。需要嚴格的 holdout validation。
    *   **模型失效:** 在極端市場情境下，歷史相關性與波動率模型可能失效，導致最佳化結果不佳。
    *   **計算效率:** 實時計算 13 個符號的動態相關性矩陣和執行最佳化可能存在性能瓶頸。
    *   **交易成本:** 頻繁調整權重可能導致較高的交易費用和滑點。

7.  **Alpha 來源:**
    *   **資源配置效率提升:** 在不同市場情境下將資金分配給預期表現最佳的策略或符號，避免資金閒置或分配不當。
    *   **風險分散與管理:** 透過動態調整權重，降低整體投資組合的波動性，提升風險調整後收益 (例如夏普比率)。
    *   **捕捉跨符號套利機會:** 雖然不是純粹的套利，但最佳化會考慮符號間的相關性，間接捕捉市場結構性變化帶來的相對價值。

### 專案方向二：適應性情境感知 Alpha 生成器 (Adaptive Regime-Aware Alpha Generator)

1.  **名稱和一句話描述:** **適應性情境感知 Alpha 生成器 (Adaptive Regime-Aware Alpha Generator)** - 讓 Calcifer/Markl 不僅選擇策略，更能根據實時市場情境，動態調整 Zangetsu V6 策略生成過程的參數，以產生更具適應性的 Alpha。

2.  **為什麼適合 j13:**
    *   **自我演化 agent 深度利用:** 直接擴展 Calcifer/Markl 的能力，從「選擇最佳策略」進化到「生成最佳策略」。
    *   **Zangetsu V6 核心強化:** 不改變 Zangetsu 的基本架構，而是讓其策略發現過程更加智慧和情境敏感。
    *   **Regime-aware 深度整合:** 將情境識別提升到策略生成層面，確保生成的策略從一開始就針對特定情境進行優化。
    *   **自主執行系統:** 策略生成和參數調整過程完全由 agent 自動完成，無需人工介入。
    *   **穩定先於擴展:** 專注於提升現有 Alpha 生成的品質和適應性，而非引入全新的 Alpha 來源。
    *   **Occam's Razor:** 透過優化現有 Zangetsu 的參數空間，而非引入全新的模型範式。

3.  **核心技術架構:**
    *   **Zangetsu V6 Arena/ELO Tournament:** 保持不變，但其輸入參數將由 Calcifer/Markl 動態提供。
    *   **AKASHA 記憶系統:** 儲存不同市場情境下，Zangetsu 策略生成參數與其產出策略表現的映射關係。
    *   **Calcifer/Markl 策略生成監督層:**
        *   **輸入:** 當前市場情境 (4h/1h/1m)、AKASHA 中儲存的歷史情境-參數-表現數據、Zangetsu Arena 的實時反饋。
        *   **核心邏輯:** Calcifer 學習在特定情境下，調整 Zangetsu V6 策略生成過程中的哪些參數 (例如：指標組合權重、入場/出場閾值範圍、止損止盈比例、K/N voting 的 K 值、Arena 中策略的生命週期等)，能最大化產生高 Alpha 策略的機率。Markl 則負責將這些調整後的參數注入 Zangetsu 的 Arena 進行策略探索。
        *   **反饋循環:**
